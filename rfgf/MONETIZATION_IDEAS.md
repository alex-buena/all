# Monetization Ideas

## Product Principle
- Keep the core party loop free and uninterrupted.
- Monetize optional variety, personalization, and convenience.
- Never block an active social moment with a hard paywall.

## Strong Candidates
- Premium themed packs (one-time purchase per pack).
- Party Pass (time-limited unlock for a single night/session).
- Plus subscription (all packs + new monthly drops).
- Creator tools (advanced custom-pack features, cloud backup, sharing links).
- Venue/event packs (bars or communities can buy branded/private packs).

## Max Rounds / Max Traits Cap (Your Idea)
- This can work as a trial mechanic, but should be used carefully.
- Risk: if a session is cut off mid-fun, users may churn instead of paying.
- Better version:
  - Unlimited core rounds and traits with the base deck.
  - Limits apply only to premium content/decks or advanced features.
  - Example: free gets base deck + rotating weekly pack; paid unlocks full library.

## Suggested Monetization Ladder
- Free:
  - Base deck
  - Unlimited rounds
  - Unlimited traits on base content
- Low-price purchase:
  - Individual themed packs
- Mid-tier:
  - Party Pass for full-night unlock
- Recurring:
  - Plus subscription for all packs and ongoing content

## When To Add Monetization
- Do not monetize before core retention is validated.
- Add first paid feature only after repeated playtest demand for more content.

## Validation Metrics Before Paywall
- Session length in social settings.
- Return rate within 7 days.
- Number of rounds per session.
- Percentage of sessions that include restart decisions and continued play.
- User requests for new topics/packs.
